4. **Task - Fix ImageViewer menu and clipboard** (v1.0.7): Fixed two issues with the SVG file viewer support<br>
   **Result**: Context menu items were not appearing reliably when right-clicking SVG files opened directly in `.jp-ImageViewer` because `isImgSvg()` relied on `app.shell.currentWidget.title.label`, which is focus-dependent and returned stale/wrong titles when multiple tabs were open. Replaced this with a DOM-based `findTabLabel()` helper that walks from the IMG to its containing `.jp-MainAreaWidget`, then uses the widget's `id` to find the matching tab via `.lm-TabBar-tab[data-id="<id>"] .lm-TabBar-tabLabel`. This links each image viewer to its own tab deterministically. Also fixed a clipboard issue where "Copy as PNG" produced a different PNG than "Save as PNG" despite both code paths going through the same `imgToPng()` -> canvas -> toBlob pipeline. Wrapped the blob in `Promise.resolve(blob)` when creating the `ClipboardItem` - this is a known workaround for Chromium-based browsers where passing a blob created in an async canvas.toBlob chain directly to ClipboardItem can cause the clipboard to receive stale or wrong content. Verified via Playwright that the context menu now shows "Copy as PNG" and "Save as PNG" items enabled on SVG files opened in the ImageViewer. Built and installed as v1.0.7.
